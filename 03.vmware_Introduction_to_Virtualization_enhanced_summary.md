## 1Ô∏è‚É£ Virtual Machine & ESXi Overview
- **VM (Virtual Machine)**: guest OS unaware it's virtualized; shares physical resources (CPU, memory, network, storage)
- **ESXi Host**: Type-1 hypervisor installed directly on hardware
- **Hypervisor Layer**: abstraction between VMs and physical resources; manages CPU/memory/network/storage access
- **Benefits**: multiple VMs on one host, independent guest OS, efficient resource utilization

## 2Ô∏è‚É£ Resource Virtualization

### a) CPU Virtualization
- Multiple VMs share CPU cores
- Oversubscription allowed: vCPUs > physical CPUs
- Analogy: restaurant seats vs daily visitors

### b) Memory Virtualization
- Virtual memory allocation can exceed physical memory
- Hypervisor maps memory dynamically as VMs request it

### c) Network Virtualization üåê
- **vSphere Standard Switch (vSS)**: virtual switch for VM communication
- VMs can communicate internally or externally
- VLANs and NIC teaming supported

### d) Storage Virtualization üíæ
- VMs use virtual disks (VMDK) stored on datastores
- Datastore types: VMFS, vSAN, Virtual Volumes
- Hypervisor redirects SCSI commands to physical storage

## 3Ô∏è‚É£ VM Components
- **Live State**: memory contents, CPU scheduling, virtual NIC activity
- **VM Files**: VMX (configuration), VMDK (virtual disk)
- Powered-off VMs: live state cleared, files remain for next boot
